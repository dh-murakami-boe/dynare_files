# Notes on Benigno and Eggertsson (2025)

This folder contains Dynare replication files for the model in:

> Benigno, P. and Eggertsson, G. (2025). "It's Baaack: The Surge in Inflation in the 2020s and the Return of the Non-Linear Phillips Curve."

The paper proposes a **nonlinear (Inverse-L shaped) New Keynesian Phillips Curve** to explain the 2020s inflation surge. The key insight is that the Phillips curve is nearly flat when the labor market is slack ($\theta \leq \theta^* \approx 1$), but becomes very steep once the labor market enters shortage territory ($\theta > \theta^*$). This nonlinearity is driven by a max operator in the wage-setting equation for newly hired workers, $w_t^{new} = \max\{w_t^{ex}, w_t^{flex}\}$, which is implemented in Dynare using the OccBin toolkit.

## Model files

### Simplified policy model (`be_simplified.mod`)

A tractable 3-equation New Keynesian system (equations 47–49 of the paper):

- **IS curve**: standard Euler equation with demand shocks.
- **Phillips curve**: piecewise-linear with two regimes separated by an output gap threshold $Y^*$.
  - *Slack regime* ($Y \leq Y^*$): flat, with slope $\kappa \approx 0.0065$.
  - *Tight regime* ($Y > Y^*$): steep, with slope $\kappa^{tight} \approx 0.0736$.
- **Taylor rule**: $\hat{i}_t = \phi_\pi \pi_t + e_t$, with $\phi_\pi = 1.5$.

Coefficients are taken from Table B (2008–2023 sample) of the paper. The regime switch is implemented using OccBin, with the constraint binding when the output gap exceeds the threshold $Y^*$.

To run this model, open MATLAB, navigate to the `model/` directory, and run:

```matlab
run_be_simplified.m
```

### Full structural model (`be_structural.mod`)

The full structural specification from Sections 3–4 of the paper:

- Rotemberg price adjustment costs.
- Search-and-matching labor market ($M_t = m_t U_t^\eta V_t^{1-\eta}$).
- Piecewise wage setting: $w_t^{new} = \max\{w_t^{ex}, w_t^{flex}\}$ implemented via OccBin. The regime switch is now directly on the wage comparison (flexible vs. existing wage) rather than the output gap proxy used in the simplified model.
- Primitive shocks: productivity ($A$), markup ($\mu$), oil/intermediate input price ($q$), demand ($g$), monetary policy ($e$), labor participation ($\chi$), separation ($\xi$), and matching efficiency ($m$).

To run this model, open MATLAB, navigate to the `model/` directory, and run:

```matlab
run_be_structural.m
```

**Requirements**: MATLAB R2025b and Dynare 6.5 (or a compatible version with OccBin support).

## Known issues

- The $Y^*$ calibration in the simplified model is approximate (set to 0.01). A more careful mapping from $\theta^* = 1$ via the structural model is needed.
- In the simplified model, OccBin can fail to converge when both demand and supply shocks are large simultaneously, due to the large discontinuity in $\kappa_\nu$ across regimes (0.0093 → 0.2742). The combined-shock scenario therefore uses a smaller supply shock.
- The sign convention for $\kappa_\nu$ follows the structural interpretation (positive pass-through from cost-push shocks to inflation), using the absolute value of the Table B estimate. See `be_paper_summary.md` for discussion.
